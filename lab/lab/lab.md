Лабораторна робота №3: SDLC
Генератор паролів

1. Планування
Мета проєкту: Створити застосунок для генерації безпечних випадкових паролів з можливістю налаштування параметрів.

Основна функціональність: Користувач повинен мати можливість генерувати надійні паролі різної довжини з вибором типів символів (великі/малі літери, цифри, спеціальні символи), копіювати згенеровані паролі та зберігати історію останніх паролів для зручності.

2. Аналіз вимог (User Stories)

Як користувач, я хочу генерувати випадковий пароль одним кліком, щоб швидко отримати безпечний пароль. ✅ (Must have)
Як користувач, я хочу налаштовувати довжину пароля (від 8 до 32 символів), щоб відповідати вимогам різних сервісів. ✅ (Must have)
Як користувач, я хочу вибирати типи символів (великі літери, малі літери, цифри, спецсимволи), щоб контролювати складність пароля.
Як користувач, я хочу копіювати згенерований пароль в буфер обміну, щоб швидко використовувати його.
Як користувач, я хочу бачити історію останніх 5 згенерованих паролів, щоб повернутися до попереднього варіанту.

4. Реалізація
Ключова функція: Генерація випадкового пароля
Псевдокод:
function generatePassword(length, options):
    // Визначаємо набори символів
    uppercaseChars = "ABCDEFGHIJKLMNOPQRSTUVWXYZ"
    lowercaseChars = "abcdefghijklmnopqrstuvwxyz"
    digitChars = "0123456789"
    specialChars = "!@#$%^&*()_+-=[]{}|;:,.<>?"
    
    // Формуємо доступний набір символів
    availableChars = ""
    if options.includeUppercase:
        availableChars += uppercaseChars
    if options.includeLowercase:
        availableChars += lowercaseChars
    if options.includeDigits:
        availableChars += digitChars
    if options.includeSpecial:
        availableChars += specialChars
    
    // Перевірка валідності
    if availableChars is empty:
        return "Error: Виберіть хоча б один тип символів"
    
    if length < 8 or length > 32:
        return "Error: Довжина має бути від 8 до 32"
    
    // Генерація пароля
    password = ""
    for i from 1 to length:
        randomIndex = getRandomNumber(0, availableChars.length - 1)
        password += availableChars[randomIndex]
    
    // Зберігаємо в історію
    addToHistory(password, length, options)
    
    return password
Пояснення: Функція приймає бажану довжину та параметри (які типи символів використовувати). Спочатку формується набір доступних символів на основі вибраних опцій, потім випадковим чином вибираються символи з цього набору для створення пароля заданої довжини. Згенерований пароль зберігається в історію.

5. Тестування
Тест-кейси:

Тест 1: Генерація пароля довжиною 12 символів з усіма типами символів → створюється пароль, що містить великі літери, малі літери, цифри та спецсимволи, загальна довжина 12 символів.
Тест 2: Спроба генерації пароля без вибору жодного типу символів → система видає помилку "Виберіть хоча б один тип символів".
Тест 3: Генерація пароля довжиною 8 символів тільки з цифр → створюється пароль з 8 цифр (наприклад, "73829156").
Тест 4: Копіювання згенерованого пароля → пароль копіюється в буфер обміну, з'являється повідомлення "Пароль скопійовано".
Тест 5: Генерація 6 паролів підряд → в історії зберігаються останні 5 паролів, найстаріший видаляється.

6. Висновки
Для цього проєкту найкраще підходить Agile-модель, оскільки застосунок є достатньо простим, але потребує можливості швидко реагувати на потреби користувачів.
Agile дозволить розробляти функціонал ітераціями: спочатку базова генерація паролів, потім додавання налаштувань, історії, оцінки надійності пароля тощо. Можна швидко тестувати кожну нову функцію з користувачами та вносити покращення.
Waterfall був би неефективним, оскільки важко передбачити всі потреби користувачів заздалегідь, а Spiral модель є занадто складною для такого невеликого проєкту. Agile забезпечує баланс між гнучкістю та структурованістю, що ідеально підходить для створення користувацьких інструментів.
